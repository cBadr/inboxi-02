# @inboxi/mta-inbound

Haraka inbound MTA. Listens on port 25, accepts catch-all mail for active Inboxi
domains (`config/host_list`), parses the MIME, and forwards each envelope
recipient to the web ingest endpoint (`/api/mail/inbound`).

## Flow

```
Internet --25--> Haraka --(rcpt_to.in_host_list: accept catch-all)
                       --> inboxi_ingest plugin
                       --> simpleParser(raw)
                       --> POST /api/mail/inbound  (x-ingest-secret)
                       --> web app persists (mailbox / anon session / catch-all)
```

## Config / env

- `MAIL_INGEST_URL` — web ingest URL (default `http://127.0.0.1:3000/api/mail/inbound`)
- `MAIL_INGEST_SECRET` — shared secret, must match the web app
- `config/host_list` — accepted domains (regenerated by the web app on domain changes)

## Run (Linux server)

```bash
pnpm install
MAIL_INGEST_SECRET=... pnpm --filter @inboxi/mta-inbound start
```

Haraka binds port 25; under systemd this is granted via `CAP_NET_BIND_SERVICE`
(see `infra/systemd/haraka-inbound.service`). Haraka is a Linux service — local
dev/tests on Windows exercise the forwarding logic via `lib/forward.js` unit
tests instead of launching the daemon.
